# Contributing Guide

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch
4. Make changes
5. Run tests
6. Submit PR

## Code Standards

- Follow RuboCop rules
- Write specs for all new code
- Use conventional commits
- Keep PRs focused and small

## Commit Messages

Format: `<type>(<scope>): <description>`

Types:
- `feat`: New feature
- `fix`: Bug fix
- `test`: Tests
- `docs`: Documentation
- `chore`: Maintenance
- `refactor`: Code refactoring

Examples:
- `feat(order): add order cancellation endpoint`
- `fix(customer): handle missing address field`
- `test: add integration tests for event flow`

## Pull Request Process

1. Update documentation if needed
2. Add tests for new functionality
3. Ensure all tests pass
4. Request review from maintainer
